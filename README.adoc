üìò AWARE-Micro ‚Äì Customized Build for Digital Phenotyping

This repository contains a customized version of the AWARE Microservice designed specifically for our Digital Phenotyping Capstone project. It integrates seamlessly with the AWARE backend running on our VMware environment and enables secure processing of sensor-based behavioral data.

*NOTE: The only 2 files modified for now are ~/aware-micro/aware-config.json and ~/aware-micro/src/main/kotlin/com/awareframework/micro/MainVerticle.kt

(aware-config.json has the database credintials, and MainVerticle.kt has the back-end RESTful APIs along with others)

------

üì¶ Setup & Installation (better on VM -> this is for server-side)
1. Clone the Repository
git clone https://github.com/PiranhaQTR/Digital-Phenotyping-Backend.git
cd Digital-Phenotyping-Backend

2. Install Dependencies
npm install

3. Run the server from the path ~/aware-micro:
./gradlew run 

------

üîå To Connect to This AWARE Backend (from client-side):

Digital-Phenotyping-Backend Aware-micro server must be running already in order to connect to it.
Set your backend URL in your environment config (from client-side, in both files: awareAPI.js and DemoScreen.js) using the IP address of the VM you're on:

AWARE_BASE_URL=http://<VM-IP>:8080

üóÑÔ∏è Viewing AWARE MySQL Database in DBeaver
You can explore all AWARE sensor tables (accelerometer, gyroscope, battery, location, notifications, etc.) directly through DBeaver, using the credentials defined in your aware-config.json file.

1. Open DBeaver:
If not installed, download it here:
https://dbeaver.io/download/

2. Launch DBeaver

3. Click Database ‚Üí New Connection

4. Select MySQL (or MariaDB)

5. Enter your credentials aware-config.json

6. Test the Connection: 
Click Test Connection.
You should get a Success window.
Then click Finish.
