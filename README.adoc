This repository contains a customized version of the AWARE Microservice designed specifically for our Digital Phenotyping Capstone project. It integrates seamlessly with the AWARE backend running on our VMware environment and enables secure processing of sensor-based behavioral data.

------

üì¶ Setup & Installation
1. Clone the Repository
git clone https://github.com/PiranhaQTR/Digital-Phenotyping-Backend.git
cd Digital-Phenotyping-Backend

2. Install Dependencies
npm install

3. Run the server from the path ~/aware-micro:
./gradlew run 

------

üîå Connect to Your AWARE Backend

This custom build expects a running AWARE server.
Set your backend URL in your environment config using the IP address of the VM you're on:

AWARE_BASE_URL=http://<VM-IP>:8080
STUDY_KEY=<your-study-key>   ---> (unused right now)

------

Your mobile application or ingestion services will push sensor data to this backend automatically once configured.

üìä Sensor Data Supported:
Category	                      Sensors Included
Device Sensors	                Accelerometer, Gyroscope, Pedometer, Location
Software Sensors	              Notifications, Screen Events, Battery Status

These are stored in AWARE‚Äôs default MySQL tables and can be queried for analytics.

üóÑÔ∏è Viewing AWARE MySQL Database in DBeaver
You can explore all AWARE sensor tables (accelerometer, gyroscope, battery, location, notifications, etc.) directly through DBeaver, using the credentials defined in your aware-config.json file.

Open DBeaver:
If not installed, download it here:
https://dbeaver.io/download/

Then:
1. Launch DBeaver

2. Click Database ‚Üí New Connection

3. Select MySQL (or MariaDB)

4. Enter your credentials:

Field	                  Value from aware-config.json
Host	                  localhost (VM IP)
Port	                  3306
Database	              aware_db 
Username	              aware_user
Password	              password

5. Test the Connection: 
Click Test Connection.
You should get a Success window.
Then click Finish.
